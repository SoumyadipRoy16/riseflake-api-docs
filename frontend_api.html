Authentication & Route Guards

Authentication Methods

MethodImplementationUsage
Email/PasswordJWT + HTTP-only CookieStandard login/register
Google OAuthFirebase Auth / OAuth2/google-callback route
LinkedIn OAuthLinkedIn API/linkedin-callback route
Microsoft OAuthMicrosoft Identity/microsoft-callback route

Route Protection Layers

  • CommonProtectRoute: Guards general user routes (requires auth token)
  • RecruiterRoute: Guards recruiter-specific routes (checks recruiter role)
  • ProtectAdminRoute: Guards admin dashboard (checks admin/sub-admin role)
  • Role-Based Redirect: /home redirects to appropriate dashboard based on user role
Token Management: JWT tokens stored in secure HTTP-only cookies. Access token refresh handled via interceptors. Logout clears all session data.

Backend Integration

API Communication

  • Base URL: /api/v1 (backend URL)
  • HTTP Client: Axios with interceptors
  • Authentication: JWT in headers + HTTP-only cookies
  • Error Handling: Interceptors for 401/403/500 errors
  • Token Refresh: Automatic refresh on 401 response

Real-Time Communication

  • Socket.io: Real-time chat & notifications
  • Events: message.send, message.receive, typing, online.status
  • Reconnection: Auto-reconnect with exponential backoff
  • Fallback: Polling if WebSocket unavailable

Third-Party Integrations

  • Firebase: Authentication, push notifications
  • Google OAuth: OAuth login integration
  • LinkedIn OAuth: Professional login
  • Microsoft Identity: Enterprise login
