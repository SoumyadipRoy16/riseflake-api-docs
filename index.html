Architecture & Request Lifecycle

The project follows a layered MVC pattern. Every HTTP request passes through the following pipeline:

Project Structure

backend-riseflake/
├── Server.js                    # Entry point, Socket.IO, cron jobs
├── api/
│   ├── v1/
│   │   ├── index.js             # Route mounting for all v1 endpoints
│   │   ├── controllers/         # Business logic handlers
│   │   ├── routes/              # Express router definitions
│   │   ├── middlewares/         # Auth, rate-limit, monitoring
│   │   └── validations/         # Input validation rules
│   └── v2/                      # Newer controller versions
└── shared/
    ├── models/                  # PostgreSQL models (raw SQL)
    └── utils/                   # DB, JWT, email, OTP, S3, Redis

Authentication

Authentication uses JWT tokens stored in signed HTTP-only cookies. The middleware reads the cookie automatically on every protected request.

Token refresh: Tokens are refreshed via POST /auth/refresh using the existing cookie. On logout, the cookie is cleared and the session is invalidated in PostgreSQL.

OAuth Providers

Google, LinkedIn, and Microsoft OAuth are supported. Admin accounts (roles 1, 6) cannot authenticate via OAuth.

Roles & Access Control

Every user has a numeric role stored in the users table. userAuthorizationHandler(role1, role2, ...) guards routes.

RoleNameDescription
1AdminFull platform access — users, jobs, subscriptions, analytics
2RecruiterCreate & manage job postings, view applications for own jobs
3Candidate / ApplicantBrowse jobs, apply, manage profile
4CollegeCollege institution account with bulk registration capabilities
5MentorMentor profile access
6Sub-AdminAdmin-level read access + limited write permissions
Routes marked Admin only require role 1 or 6. Routes marked Recruiter require role 2. Most profile/application routes are role 3 (candidate).

Standard Error Responses

All endpoints return JSON. Errors follow a consistent shape:

{
  "status": false,
  "message": "Human-readable error description"
}
HTTP CodeMeaningCommon Cause
400Bad RequestMissing or invalid request body / query parameters
401UnauthorizedMissing or expired JWT cookie
403ForbiddenAuthenticated but insufficient role
404Not FoundResource with given ID/slug does not exist
409ConflictDuplicate email, mobile number, or resource
429Too Many RequestsRate limiter triggered (express-rate-limit + Redis)
500Internal Server ErrorUnhandled exception — check server logs

WebSocket Events

The server uses Socket.IO 4 with a Redis adapter for horizontal scalability. Connect to the server root with your JWT cookie present.

EventDirectionPayload
new_messageServer → Client{ message_id, sender_id, content, created_at }
message_seenServer → Client{ message_id, seen_by }
message_deletedServer → Client{ message_id }
connection_requestServer → Client{ from_user, connection_id }
connection_acceptedServer → Client{ connection_id }
notificationServer → Client{ type, payload }
typingClient → Server{ to_user_id, is_typing }

Setup Guide

Prerequisites

Node.js 18+, PostgreSQL 14+, MongoDB, Redis

1. Install

git clone <repository-url>
cd backend-riseflake
npm install

2. Configure

cp .env.example .env
# Edit .env with your credentials

3. Initialize Database

node shared/models/init.js

4. Run

# Development
npm run dev

# Production
npm start
Cron jobs run automatically on server start: session cleanup daily at 3:00 AM, and top-profiles snapshot every hour.

Scheduled Tasks

ScheduleTask
0 3 * * *Delete sessions older than 90 days
0 * * * *Sync top-profiles snapshot to cache
