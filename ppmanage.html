Profile Photo
Management & Approval

Comprehensive backend system for user profile photo upload, validation, storage, and admin review workflow. Handles image processing, content moderation, and account verification requirements.

Photo Upload Workflow

The upload process follows a multi-stage pipeline: client upload → server validation → image processing → storage → optional admin review → profile update.

Key Points: Photos are stored with version control. Previous photos are archived, not deleted. Approved photos become the active profile image. Rejected photos remain in the user's gallery for re-submission after corrections.

Admin Approval System

All profile photos go through an optional admin review queue. The system flags suspicious or policy-violating content for manual verification.

Security & Validation

File Upload Security

  • File Size: Maximum 5 MB per image
  • Formats Allowed: JPEG, PNG only (MIME: image/jpeg, image/png)
  • Dimensions: Minimum 200x200px, maximum 5000x5000px
  • File Validation: Magic number check (no disguised files)
  • Rate Limiting: Max 10 uploads per hour per user
  • Virus Scan: Optional ClamAV integration for production

Storage & Access Control

  • Photos stored with randomized filenames (UUID prefix)
  • S3/R2 bucket configured for public read (CDN delivery)
  • Original file not stored; only processed versions
  • EXIF metadata stripped before upload
  • Soft-delete: photos marked as archived, not permanently deleted

Content Moderation

Auto-flagging criteria: Dimensions outside range · Corrupt metadata · Duplicate detection (hash matching) · Manual admin review required for borderline cases

Storage Strategy

Cloud Storage (AWS S3 / Cloudflare R2)

Profile photos are stored in a dedicated S3 bucket or R2 instance with the following structure:

s3://riseflake-photos/
├── profiles/
│   ├── {user_id}/
│   │   ├── {uuid}-original.jpg      (archived, not public)
│   │   ├── {uuid}-512x512.jpg       (main profile photo)
│   │   └── {uuid}-256x256.jpg       (thumbnail)
│   ├── ...
├── temp/
│   ├── {upload_session_id}/         (temporary during upload)
│   └── ...
└── archive/
    └── {user_id}-{timestamp}/       (monthly backups)
CDN Strategy: CloudFront or similar CDN fronts the S3 bucket for faster global delivery. Image URLs are publicly accessible (no authentication needed for viewing).

Local Processing

All image processing happens on the backend server using Sharp library:

// Pseudocode
const image = await sharp(uploadBuffer)
  .resize(512, 512, { fit: 'cover' })
  .jpeg({ quality: 85, progressive: true })
  .toBuffer();

const thumb = await sharp(uploadBuffer)
  .resize(256, 256, { fit: 'cover' })
  .jpeg({ quality: 80, progressive: true })
  .toBuffer();

Environment Variables

# Profile Photo Storage
PHOTO_MAX_FILE_SIZE=5242880               # 5 MB in bytes
PHOTO_ALLOWED_FORMATS=jpeg,png
PHOTO_MIN_DIMENSIONS=200
PHOTO_MAX_DIMENSIONS=5000
PHOTO_THUMB_SIZE=256

# Rate Limiting
PHOTO_UPLOAD_RATE_LIMIT=10                # uploads per hour
PHOTO_UPLOAD_TIME_WINDOW=3600             # seconds

# S3/R2 Configuration
S3_BUCKET_PHOTOS=riseflake-photos
S3_REGION=us-east-1
R2_BUCKET_NAME=riseflake-photos
R2_ENDPOINT=https://r2-endpoint.example.com

# Processing
SHARP_QUALITY_MAIN=85                     # JPEG quality for main image
SHARP_QUALITY_THUMB=80                    # JPEG quality for thumbnail

# Admin Review
PHOTO_AUTO_APPROVE_ENABLED=true           # Skip queue if no flags
PHOTO_REQUIRE_ADMIN_APPROVAL=false        # Force review for all
PHOTO_CONTENT_MODERATION_ENABLED=true     # Use auto-flagging

# Retention
PHOTO_ARCHIVE_RETENTION_DAYS=90
PHOTO_REJECTED_RETENTION_DAYS=30
PHOTO_TEMP_CLEANUP_HOURS=24
